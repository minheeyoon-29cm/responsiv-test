29CM 반응형 의사결정을 위한 프로토타입

이 사이트는 데스크탑에서 보시는 걸 권장드립니다.

이 사이트는 앞으로 적용될 29CM의 반응형 디자인 전략을 미리 확인할 수 있는 프로토타입으로, 각 페이지의 반응형 분기 선택에 대한 디자인 및 개발 의사결정을 요약한 페이지입니다.

데스크탑 화면 스타일 (2560px 이상)

@media only screen and (min-width: 2560px) {
.container { max-width:2560px; }
}

2560px 이상의 넓은 화면에서 적용됩니다. 주로 데스크탑과 같은 큰 화면을 기준으로, 콘텐츠가 2560px의 최대 너비 내에서 중앙 정렬됩니다.

design token : container-max-width-2560

데스크탑 화면 스타일 (1281px 이상)

@media only screen and (min-width: 1281px) {
.container { max-width:1280px; }
}

이 부분은 1281px 이상의 넓은 화면에서 적용됩니다. 주로 데스크탑과 같은 큰 화면을 기준으로, 콘텐츠가 1280px의 최대 너비 내에서 중앙 정렬됩니다.

design token : container-max-width-1280

1025px ~ 1280px 사이의 데스크탑 화면 (포인터 디바이스)

@media only screen and (min-width: 1025px) and (max-width: 1280px) and (pointer: fine) {
.container { max-width: 1280px; margin: 0 auto; }
}

1025px에서 1280px 사이의 포인터 디바이스 (주로 데스크탑)에서 적용됩니다. 이 구간에서는 화면 너비가 1025px에서 1280px 사이에 맞춰 자연스럽게 줄어들며, 콘텐츠 너비는 최대 1280px까지 확장되지만, 화면에 맞게 조정되어 가로 스크롤이 생기지 않습니다.

design token : container-max-width-1280

모바일 화면 스타일 (1024px 이하)

@media only screen and (max-width: 1024px) {
.container { width: 100%; max-width: 700px; }
}

1024px 이하의 작은 화면(주로 모바일)에서 적용됩니다. 콘텐츠의 너비가 100%로 확장되어 화면 전체를 채우며, 최대 너비는 700px로 제한됩니다.

design token : container-max-width-700

터치 디바이스 스타일 (1280px 이하의 터치 기반 디바이스)

@media only screen and (max-width: 1280px) and (pointer: coarse) {
.container { max-width: 700px; margin: 0 auto; }
}

1280px 이하의 터치 디바이스(태블릿 또는 모바일)에서 적용되며, 콘텐츠의 최대 너비를 700px로 제한하여 터치 기반의 편리한 인터페이스를 제공합니다.

design token : container-max-width-700

각 분기 선택의 이유

유지보수 효율성
1280px, 1024px, 700px과 같은 주요 분기는 여러 해상도에서 디자인을 재사용할 수 있도록 해 줍니다. 개발자와 디자이너가 한 번 설정해 놓으면, 새로운 화면 크기나 디바이스가 추가되더라도 기존 설정에 따라 대응할 수 있기 때문에 유지보수 공수가 크게 줄어듭니다.

디자이너의 입장에서도 디자이너가 반응형 디자인을 준비할 때, Figma의 오토 레이아웃과 그리드 시스템을 잘 활용하면 추가적인 디자인 작업이 거의 필요하지 않습니다. 예를 들어, 1280px와 1024px 사이의 차이(256px)만 신경 써서 내부 콘텐츠 배치를 디자인하면, 나머지는 브라우저가 자동으로 조정해 줍니다.

즉, 모듈의 비율과 그리드 구조만 고려하여 디자인을 준비하면 다양한 해상도에 자연스럽게 대응할 수 있습니다. 핵심은 1024px 이하의 해상도까지도 자연스럽게 맞춰지도록 설계하되, 별도로 1024px 전용 디자인을 준비할 필요가 없다는 점입니다. 디자이너는 1280px(데스크탑)과 700(모바일)px 단 2벌의 디자인만 고려하면 됩니다.


일관된 화면 규칙성
각 분기를 명확하게 정의함으로써, 디자이너와 개발자는 특정 해상도와 디바이스 유형을 일관되게 지원할 수 있습니다. 특히, 1280px과 1024px 분기를 기본으로 작업하면, 화면 크기가 다양해져도 새로운 레이아웃을 추가할 필요 없이 효율적인 유지보수가 가능합니다.


다양한 디바이스 경험 고려
모바일, 태블릿, 데스크탑 등 각 디바이스는 서로 다른 사용 방식을 가지고 있습니다. 29CM 사용자들은 다양한 기기를 통해 접속하기 때문에, 디바이스에 맞는 경험을 제공하면서도 일관성을 유지하는 것이 중요합니다. 터치와 포인터 디바이스를 구분하여 반응형 디자인에서 다양한 사용자 경험(UX) 요구를 전략적으로 반영하고자 합니다.


확장성과 예외 케이스 보완
2560px와 같은 고해상도 분기를 추가함으로써, 일반적인 화면 환경을 넘어서는 디바이스에서도 사용자의 요구를 충족시킬 수 있습니다. 특히, 고해상도 모니터를 사용하는 사용자에게 최적화된 경험을 제공하며, 다양한 화면 크기에서 일관된 품질을 유지합니다.
또한, 29CM의 대형 기획전과 같이 시각적 임팩트가 중요한 상황에서도 다양한 디자인 표현을 지원할 수 있어, 다채로운 비주얼 플레이와 차별화된 경험을 제공할 수 있습니다.


1024px 해상도를 고려해야 하는 이유

적은 사용자 수와 점차 사라지고 있는 1024px 해상도를 여전히 고려해야 하는 이유에 대해 생각해보겠습니다.

미래는 예측할 수 없기 때문에 디자인에서 유연성과 확장성을 확보하는 것이 매우 중요합니다. 1024px 이하의 해상도를 고려하는 것은 앞으로 변화하는 다양한 해상도에 대응할 수 있는 준비를 의미합니다.

예를 들어, 한때 모바일의 최소 기준 해상도였던 320px이 360px로 상향되었을 때 많은 디자이너들이 더 넓어진 작업 영역을 반겼습니다. 하지만 최근에는 갤럭시 폴드4와 같은 345px 가로폭을 가진 디바이스도 등장하면서, 또다시 최소 해상도에 맞춰야 하는 도전이 생겼습니다.

제 의견은 1024px 전용 디자인을 반드시 준비하자는 것이 아닙니다. 대신, 모든 해상도에서 유연하게 대응할 수 있는 레이아웃을 설계하는 것이 중요하다는 것입니다. 최소한 1024px까지는 대응 가능한 디자인을 만드는 것이 여전히 유효한 접근이라는 의견입니다.

사례 보기

에어비앤비의 제품 상세 페이지(PDP)는 1024px 이하 해상도에서도 문제없이 표시되도록 설계되었습니다.

쇼피파이 또한 1024px 이하의 해상도에서 문제없이 보여지도록 디자인되어 있습니다.

디자인과 개발 시 고려할 것

우리의 반응형 전략에 맞춰 화면의 크기와 디바이스의 입력 방식에 따라 자동으로 최적화된 레이아웃이 설정됩니다. 따라서, 이제는 모바일 기기와 데스크탑의 각각 다른 사용 방식을 고려한 인터랙션 설계에 집중하는 것이 중요합니다.

데스크탑
  • 그리드 시스템: 화면 크기에 따라 컬럼 수를 조정하세요.
  • 콘텐츠의 너비: 자동으로 줄어들 수 있는 너비와 여백을 설정하세요.
  • 큰 화면과 포인터(마우스) 기반 인터랙션을 사용하는 환경이므로, 클릭과 마우스 오버(hover) 등의 조작에 반응할 수 있도록 세밀한 인터랙션을 설계합니다.
모바일 기기
  • 작은 화면에서 터치 기반 인터랙션을 사용하므로, 손가락으로 쉽게 터치할 수 있는 넓은 버튼과 스크롤을 통해 쉽게 탐색할 수 있는 디자인을 고려합니다.
