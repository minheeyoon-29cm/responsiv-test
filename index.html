29CM 반응형 의사결정을 위한 프로토타입


적은 사용자 수와 점차 사라지고 있는 1024px 해상도를 여전히 고려해야 하는 이유에 대해 생각해보겠습니다.

미래는 예측할 수 없기 때문에 디자인에서 유연성과 확장성을 확보하는 것이 매우 중요합니다.
1024px 이하의 해상도를 고려하는 것은 앞으로 변화하는 다양한 해상도에 대응할 수 있는 준비를 의미합니다.

예를 들어, 한때 모바일의 최소 기준 해상도였던 320px이 360px로 상향되었을 때 많은 디자이너들이 더 넓어진 작업 영역을 반겼습니다. 하지만 최근에는 갤럭시 폴드4와 같은 345px 가로폭을 가진 디바이스도 등장하면서, 또다시 최소 해상도에 맞춰야 하는 도전이 생겼습니다.

제 의견은 1024px 전용 디자인을 반드시 준비하자는 것이 아닙니다.
대신, 모든 해상도에서 유연하게 대응할 수 있는 레이아웃을 설계하는 것이 중요하다는 것입니다. 최소한 1024px까지 대응 가능한 디자인을 만드는 것이 여전히 유효한 접근이라는 것입니다.



해외 사례를 살펴보겠습니다.


에어비앤비의 제품 상세 페이지(PDP)는 1024px 이하 해상도에서도 문제없이 표시되도록 설계되었습니다.


쇼피파이 또한 1024px 이하의 해상도에서 문제없이 보여지도록 디자인되어 있습니다.


이와 같이 작업하는 것이 디자이너에게 큰 공수 부담을 주는 일일까요? 그렇지 않습니다.

반응형 디자인을 준비할 때 Figma의 오토 레이아웃과 그리드 시스템을 잘 활용하면, 각기 다른 해상도에 맞춘 디자인을 추가로 준비할 필요가 없습니다.
예를 들어 1280px와 1024px 사이 256px 사이의 내부 콘텐츠 기준만 고려하여 디자인하면, 대부분의 조정은 브라우저가 자동으로 처리하게 됩니다.
이 프로토타입 사이트를 구성한 디자인을 보면 그렇게 큰 공수가 들지 않는다는 것을 알 수 있을 것 입니다.

즉, 모듈을 설계할 때 각 콘텐츠의 비율과 그리드 구성만 신경 써서 작업하면, 반응형 웹사이트는 자연스럽게 다양한 해상도에 대응할 수 있습니다.

중요한 점은 1024px 이하의 해상도에서도 자연스럽게 레이아웃이 맞춰지도록 설계하되, 별도의 디자인 파일을 추가 준비할 필요는 없다는 것 입니다.

데스크탑 화면 스타일 (1281px 이상)

@media only screen and (min-width: 1281px) {
.container { max-width:1280px; }
}

이 부분은 1281px 이상의 넓은 화면에서 적용됩니다. 주로 데스크탑과 같은 큰 화면을 기준으로, 콘텐츠가 1280px의 최대 너비 내에서 중앙 정렬됩니다.

1025px ~ 1280px 사이의 데스크탑 화면 (포인터 디바이스)

@media only screen and (min-width: 1025px) and (max-width: 1280px) and (pointer: fine) {
.container {
max-width: 1280px;
margin: 0 auto; }
}

1025px에서 1280px 사이의 포인터 디바이스 (주로 데스크탑)에서 적용됩니다. 이 구간에서는 화면 너비가 1025px에서 1280px 사이에 맞춰 자연스럽게 줄어들며, 콘텐츠 너비는 최대 1280px까지 확장되지만, 화면에 맞게 조정되어 가로 스크롤이 생기지 않습니다.

  • max-width: 1280px; 으로 화면이 1280px보다 좁아질 경우 자동으로 화면 너비에 맞춰 줄어들기 때문에 가로 스크롤이 생기지 않습니다.

    • 모바일 화면 스타일 (1024px 이하)

    @media only screen and (max-width: 1024px) {
    .container {
    width: 100%;
    max-width: 700px; }
    }

    1024px 이하의 작은 화면(주로 모바일)에서 적용됩니다. 콘텐츠의 너비가 100%로 확장되어 화면 전체를 채우며, 최대 너비는 700px로 제한됩니다.

    터치 디바이스 스타일 (1280px 이하의 터치 기반 디바이스)

    @media only screen and (max-width: 1280px) and (pointer: coarse) {
    .container {
    max-width: 700px;
    margin: 0 auto; }
    }

    1280px 이하의 터치 디바이스(태블릿 또는 모바일)에서 적용되며, 콘텐츠의 최대 너비를 700px로 제한하여 터치 기반의 편리한 인터페이스를 제공합니다.

    이와 같이, CSS는 4개의 분기로 구성되어 다양한 화면 크기와 디바이스에 맞게 레이아웃을 자동으로 조정합니다.

    • CSS 분기 4개는 코드에서 화면 크기와 입력 방식(터치 또는 포인터)에 맞춰 자동으로 레이아웃을 조정합니다.
    • 하지만 핵심은, 1024px와 1280px이 두 가지 분기만 기억하면 된다는 것입니다.
    • 디자인 작업은 2가지 주요 화면 크기에 맞추면 충분합니다. 개발 시 CSS가 해당 화면에 맞는 스타일을 적용하여 디자이너의 작업을 그대로 반영합니다.
    • 1280px 이상 화면에서는 콘텐츠가 최대 1280px 너비로 표시됩니다. 1280px (좌우 패딩 50px로 실제 작업 영역은 1180px): 넓은 화면을 위한 데스크탑 디자인
    • 1024px 이하 화면에서는 콘텐츠가 최대 700px로 맞춰집니다. 700px (좌우 패딩 20px로 실제 작업 영역은 660px): 좁은 화면을 위한 모바일 및 태블릿 디자인
    • 예외적으로 container 영역이 100%로 확장되는 매우 넓은 화면을 작업할 경우도 1024px의 분기만 고려하여 작업하면 됩니다.

    최종적으로 디자이너는 1280px과 700px의 두 가지 레이아웃에 맞춰 작업을 진행하면, 각기 다른 화면 크기와 디바이스에서 자동으로 조정되는 반응형 웹 디자인을 완성할 수 있습니다.

    고려할 것

    우리의 반응형 전략에 맞춰 화면의 크기와 디바이스의 입력 방식에 따라 자동으로 최적화된 레이아웃이 설정됩니다. 따라서, 이제는 모바일 기기와 데스크탑의 각각 다른 사용 방식을 고려한 인터랙션 설계에 집중하는 것이 중요합니다.


    데스크탑
    • 그리드 시스템: 화면 크기에 따라 컬럼 수를 조정하세요.
    • 콘텐츠의 너비: 자동으로 줄어들 수 있는 너비와 여백을 설정하세요.
    • 큰 화면과 포인터(마우스) 기반 인터랙션을 사용하는 환경이므로, 클릭과 마우스 오버(hover) 등의 조작에 반응할 수 있도록 세밀한 인터랙션을 설계합니다.
    모바일 기기
    • 작은 화면에서 터치 기반 인터랙션을 사용하므로, 손가락으로 쉽게 터치할 수 있는 넓은 버튼과 스크롤을 통해 쉽게 탐색할 수 있는 디자인을 고려합니다.
